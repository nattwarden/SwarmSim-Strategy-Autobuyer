# SwarmSim Strategy Autobuyer 0.12.2

## Verified root cause

`buildLaboratoryArmySnapshot()` currently:

- scans every `game.unitlist()` entry on the `territory` tab
- includes the `territory` resource
- stores `getScenarioCanonicalUnitKey(unit)` as live `unitId`
- produces ids such as `unit swarmling v`
- marks all territory-tab rows as HoM affected

Phase 1 instead uses canonical base ids:

```js
const LABORATORY_PHASE1_AFFECTED_UNIT_IDS = [
  "swarmling",
  "stinger",
  "spider",
  "mosquito",
  "locust",
  "roach",
  "giantspider",
  "centipede",
  "wasp",
  "devourer",
  "goon",
];
```

The mismatch caused the first real-save snapshot to contain zero affected counts and `affectedTerritoryPerSecondTotal = 0` even though total territory production was nonzero and enormous.

## 1. Canonical affected-id resolver

Implement one shared resolver, for example:

```js

  game,
  warnings,
  formulaStatuses

```

### Primary source

Read the runtime House of Mirrors definition from:

```js
getLaboratoryHouseOfMirrorsAbility(game)
```

Inspect its `effect` array.

Select only effects whose type is `compoundUnit`.

Extract canonical base unit ids from:

```js
effect.unit?.name
effect.unittype
```

Normalize only to base runtime names.

### Expected set

Compare the runtime effect set with `LABORATORY_PHASE1_AFFECTED_UNIT_IDS`.

Requirements:

- exactly 11 unique ids
- exact set parity
- no `territory`
- no id beginning with `unit `
- no tier suffix
- no display label used as action identity

### Fallback

- use the fixed source-verified list
- add a clear warning

- do not invent ids

- export both sets
- mark mismatch
- do not present the result as globally verified
