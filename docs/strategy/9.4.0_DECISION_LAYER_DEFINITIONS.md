## Resolving the diagnosis-report "contradiction"

The lab report showed both `M2 winner = none` (live) and `global evaluator winner = Clone Ramp`
(order test). Both are the **same function** `buildUnifiedPurchaseEvaluator`, fed **two different
candidate lists**:

- **Live runtime** feeds `unifiedPurchaseProposalState.proposals` — a **curated 4-lane set** built by
  `buildUnifiedPurchaseProposals` ([user.js:18968](../../src/SwarmSim-Strategy-Autobuyer.user.js#L18968)):
  Engine (Expansion/Hatchery, key `engine`), Energy production (key `energy`),
  **Meat = `buildMeatGoalPlan().actionUnit` only** (key `meat`), Territory (key `territory`).
  On the pathological save every one is HOLD → `winner = none`.
- **The order test (TRACK 2)** fed `inspector.laneCandidates` — a **27-entry display array** populated
  by `addLaneCandidate` across *all* planners (incl. Clone Ramp BUY, Meat parent-step, Twin,
  Ability). Given that list the evaluator picked `Clone Ramp`.

**Neither list contains the Meat parent-step that actually executes as a rankable BUY winner.** The
parent-step / unlock / twin planners buy through the legacy path and never enter the curated proposal
set. That is the core defect DEL 2 repairs.

## Comparability contract (how a candidate becomes rankable)

- `evaluatePurchaseCandidate(candidate)` ([user.js:2198](../../src/SwarmSim-Strategy-Autobuyer.user.js#L2198))
  reads `candidate.raw` and emits `sharedOutcome` (`whole-economy-outcome.v2`).
- `sixDomainComparableValue(outcome)` ([user.js:4951](../../src/SwarmSim-Strategy-Autobuyer.user.js#L4951))
  returns COMPARABLE when finite, by priority:
  1. `sharedOutcome.etaImprovementSeconds` → basis `milestone-eta-seconds`
  2. `sharedOutcome.projectedMilestoneProgressDelta` → basis `same-unit-milestone-progress-delta`
  3. `comparability.commonValue` → basis `versioned-source-or-runtime-derived-common-value`
- Otherwise the domain is **UNRANKED** with a named `missingConversions` entry.

So to make a Meat planner candidate rankable it must carry `raw.etaImprovementSeconds` **or**
`raw.projectedMilestoneProgressDelta` against the active milestone — the same metric basis as the
other lanes in the cycle.

## Target-identity split (DEL 1 defect)

`captureSixDomainDecisionSnapshot` ([user.js:5056](../../src/SwarmSim-Strategy-Autobuyer.user.js#L5056))
sets `activeTarget = strategyContext.activeTarget || selectedMainAction?.target || smartFocus || …`,
and `smartRunOnce` passes `activeTarget: preExecutionMainAction?.target || smartFocus`. When the M2
winner is `none`, `preExecutionMainAction` is null, so `activeTarget` falls back to **`smartFocus`
(the long-term / UI focus = "territory")** while `activeMilestone` comes from
`getCurrentStrategyGoal` (= "lesser hive mind"). DEL 1 makes `activeTarget` follow the current
**executable milestone** target, distinct from long-term direction and UI focus.
