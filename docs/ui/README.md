This folder contains the active delivery layer for the Council Chamber UI.

## Reading order

1. `../BOOK-06-council-chamber-ui-and-experience.md` — durable vision
2. `COUNCIL_UI_CURRENT_STATUS.md` — active milestone and exact next action
3. `COUNCIL_UI_PRODUCT_DELIVERY_RUNBOOK.md` — milestone gates and checklists
4. `COUNCIL_UI_DATA_MAP.md` — presentation needs mapped to runtime truth
5. `reference/swarmbot-council-chamber-north-star.png` — visual north star

## Boundaries

- UI work does not change planner intent or hard safety defaults.
- Runtime truth remains in the canonical script and configured runtime source.
- Reference artwork is not automatically a production runtime asset.
- Generated assets need provenance, optimization, and visual acceptance before
  they enter the product.
